371. Sum of Two Integers

不用 + / -，用 XOR 表示不含進位的加法，用 AND + 左移表示進位。Python 需要把每次結果限制在 32-bit，才能模擬 LeetCode 常見的 signed int 行為。

Bit addition stepper

每一輪把加法拆成 sum without carry 與 carry。當 carry 變成 0，32-bit pattern 就是答案。

step 0 / 0
current a as signed 0
current b as signed carry 0
XOR 相同 bit 抵消，不同 bit 留下；這就是「不處理進位」的加法。 a ^ b
AND + shift 兩邊都是 1 的位置會產生進位，進位要加到左邊一格。 (a & b) << 1
MASK 只保留低 32 bits，避免 Python 的無限長整數讓負數帶著無限個 leading 1。 & 0xffffffff

Unsigned / Signed 範圍

Interpretation Range Meaning
Unsigned 32-bit 0 到 2^32 - 1 只把 32 個 bits 當作非負整數看。
Signed 32-bit -2^31 到 2^31 - 1 最高位是 sign bit。0 表示非負，1 表示負數。
同一串 32-bit pattern 可以有兩種解讀。
11111111 11111111 11111111 11111111 當 unsigned 是 4294967295，但當 signed two's complement 是 -1。

為什麼要用 MASK

Python 的 int 沒有固定 32-bit 上限，負數做 bit operation 時概念上有無限多個 leading 1。這跟 32-bit int 不一樣，所以每輪都用 MASK = 0xffffffff 把結果裁回低 32 bits。
負數特別需要 mask：如果沒有 mask，像 -1 的 bit pattern 不是只有 32 個 1，而是可以一直往左延伸的 1。carry 可能無法照 32-bit 規則收斂到 0。

最後如何還原負數

經過 mask 後，Python 看到的 a 是一個非負整數，也就是 unsigned interpretation。因此 a <= MAX_INT 不是在問「原本是不是負數」，而是在檢查 32-bit sign bit 是否為 0。
還原規則

MAX_INT = 0x7fffffff = 2^31 - 1。如果 masked value 小於等於它，最高位是 0，可以直接回傳。

如果 masked value 大於 MAX_INT，代表最高位是 1，在 signed 32-bit 中是負數。要轉回 Python 的負數值，就減掉 2^32。

例如 0xffffffff 是 4294967295，還原成 4294967295 - 4294967296 = -1。

程式碼

class Solution:
    def getSum(self, a: int, b: int) -> int:
        MASK = 0xffffffff
        MAX_INT = 0x7fffffff

        while b != 0:
            carry = ((a & b) << 1) & MASK
            a = (a ^ b) & MASK
            b = carry

        return a if a <= MAX_INT else a - (2**32)
Time / Space Complexity
Time Complexity O(1)
Space Complexity O(1)

題目通常以 32-bit signed int 模擬，所以迴圈最多把 carry 往左推過固定 32 個 bit，時間是 O(32)，寫成 O(1)。

只使用固定數量變數：MASK、MAX_INT、carry、a、b。
