Binary Tree Traversal

Pre-order、In-order、Post-order 都是 DFS。差別只在「visit root」這件事發生在遞迴左子樹與右子樹的哪個位置。

核心差異

Time / Space Complexity
Time Complexity O(n)
Space Complexity O(h)

三種 traversal 都會 visit 每個節點一次，所以時間複雜度是 O(n)。

遞迴 stack 的最大深度等於樹高 h，所以額外空間是 O(h)。平衡樹時 h = log n；最壞退化成鏈狀樹時 h = n。

如果把輸出 answer 也算入空間，answer 需要 O(n)。面試通常把輸出空間和額外工作空間分開討論。

程式碼對照

def dfs(node):
    if not node: return
    visit(node)        # pre-order
    dfs(node.left)
    visit(node)        # in-order
    dfs(node.right)
    visit(node)        # post-order
