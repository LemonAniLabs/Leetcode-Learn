113. Path Sum II

用 DFS backtracking 找所有 root-to-leaf path。每次進入節點就加入 `path`，離開節點前一定要 `path.pop()` 還原狀態。

DFS path tracking

藍色是目前 path，橘色是正在處理的節點，綠色代表找到符合 targetSum 的 root-to-leaf path。

核心不變量

path = root 到目前 node 的路徑
curSum = sum(path)
只有 leaf 且 curSum == targetSum 才加入 ans

容易漏掉：找到答案時要用 `ans.append(path[:])`。如果直接 append `path`，後面 backtracking 的 `path.pop()` 會修改同一個 list reference，最後答案會壞掉。
Time / Space Complexity
Time Complexity O(n)
Space Complexity O(h)

DFS 會拜訪每個節點一次，所以 traversal 本身是 O(n)。找到答案時複製 path 需要 O(h)，總輸出成本取決於答案數量。

遞迴 stack 和目前 path 最多都是樹高 h，所以額外工作空間是 O(h)。如果把輸出 ans 算進去，最壞需要 O(number_of_paths * h)。

狀態

path []
ans []

程式碼對照

if not node: return
path.append(node.val)
curSum += node.val
if not node.left and not node.right:
    ans.append(path[:])
dfs(node.left, curSum)
dfs(node.right, curSum)
path.pop()
