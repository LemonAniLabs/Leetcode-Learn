2602. Minimum Operations to Make All Array Elements Equal

對每個 query q，把排序後的 nums 用二分搜切成左側小於 q、右側大於等於 q，兩邊成本都能用 prefix sum O(1) 算出來。

Sorted nums + prefix sum + bisect

藍色左側需要增加到 q，橘色右側需要減少到 q。調整 q 可以看到分割點 p 如何移動。

q = 7
分割點 p 0
left cost 0
right cost 0
answer 0

公式

`p` 是第一個 `nums[p] >= q` 的位置。排序後，左邊一定都小於 q，需要往上補；右邊一定都大於等於 q，需要往下削。`prefix[p]` 是左側總和，`prefix[n] - prefix[p]` 是右側總和。

容易漏掉：每個 query 後 nums 會 reset，但我們不需要真的修改 nums。排序和 prefix sum 只針對原始 nums 做一次，之後每個 query 都用同一份資料計算。
Time / Space Complexity
Time Complexity O((n + m) log n)
Space Complexity O(n)

排序 nums 需要 O(n log n)。每個 query 用 `bisect_left` 找分割點，需要 O(log n)，共 m 個 query。

prefix sum 陣列長度是 n + 1，所以額外空間是 O(n)。

狀態

sorted nums []
prefix []

程式碼

from bisect import bisect_left
from itertools import accumulate
from typing import List

class Solution:
    def minOperations(
        self,
        nums: List[int],
        queries: List[int]
    ) -> List[int]:
        nums.sort()
        n = len(nums)
        prefix = [0] + list(accumulate(nums))
        ans = []

        for q in queries:
            # p: first index where nums[p] >= q
            p = bisect_left(nums, q)

            # left side nums[0:p] are smaller than q.
            # They need to be increased to q.
            cost_left = q * p - prefix[p]

            # right side nums[p:n] are greater than or equal to q.
            # They need to be decreased to q.
            cost_right = (prefix[n] - prefix[p]) - q * (n - p)

            ans.append(cost_left + cost_right)

        return ans
