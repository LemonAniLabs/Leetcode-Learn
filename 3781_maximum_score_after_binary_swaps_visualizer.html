3781. Maximum Score After Binary Swaps

每個 `1` 最後可以透過不斷和左邊的 `0` 交換，搬到目前前綴裡某個位置。從左到右掃描時，每遇到一個 `1`，就在目前看過但尚未被選走的 nums 裡挑最大值。

掃描前綴與 Max Heap

每一步先把 `nums[i]` 放進 heap。若 `s[i] == '1'`，代表現在必須選一個前綴候選貢獻分數，所以 pop 最大值。

heap 裡的候選 用 Python 實作時存的是負數
已被 1 選走的值 加總成 ans
步驟 0 / 6
目前 i -
heap top -
ans 0

為什麼可以貪心

看到第 i 格時：
heap = 前綴 nums[0..i] 中還沒被選走的值
如果 s[i] == '1'：選 heap 最大值加入 ans

Time / Space Complexity
Time Complexity O(n log n)
Space Complexity O(n)

每個 index 都會被掃描一次，所以外層迴圈是 O(n)。每次都會執行一次 `heappush`，成本是 O(log n)。

只有當 `s[i] == '1'` 時才會 `heappop`，最多 pop n 次，每次也是 O(log n)。因此總時間是 O(n log n)。

heap 最多會暫存所有尚未被選走的前綴元素，最壞情況大小是 n，所以額外空間是 O(n)。

變數狀態

push -
s[i] -
pop -
heap size 0

程式碼對照

heap = []
ans = 0
for i in range(len(nums)):
    heapq.heappush(heap, -nums[i])
    if s[i] == '1':
        ans += -heapq.heappop(heap)
return ans

執行紀錄

i push s[i] pop ans
