2903. Find Indices With Index and Value Difference I

固定右端 `j` 往右掃，合法左端只會來自 `i <= j - indexDifference`。維護合法候選中的 prefix min / prefix max，就能 O(n) 找答案。

Prefix min / max candidates

藍色是目前可和 j 配對的合法 prefix 候選；綠色標記 min_idx，橘色標記 max_idx，紅色是目前 j。

核心不變量

i = j - indexDifference
legal candidates: index <= i
check nums[j] - min_value or max_value - nums[j]

容易漏掉：這不是一般 sliding window。候選集合只會往右擴張，不需要移除舊 index；我們只要維護目前合法 prefix 裡的最小值位置和最大值位置。
Time / Space Complexity
Time Complexity O(n)
Space Complexity O(1)

每個 j 掃一次，每次只更新 min_idx / max_idx 並做常數次比較，所以時間是 O(n)。

只保存兩個候選 index 和結果狀態，不需要額外資料結構，所以空間是 O(1)。

程式碼對照

min_idx = max_idx = 0
for j in range(len(nums)):
    i = j - indexDifference
    if nums[i] < nums[min_idx]: min_idx = i
    if nums[i] > nums[max_idx]: max_idx = i
    if abs(nums[j] - nums[min_idx]) >= valueDifference
    if abs(nums[j] - nums[max_idx]) >= valueDifference
return [-1, -1]
